## 二、端口被封的真实原因，以及隐藏的坑

端口被封，一般分两种情况：

**第一种：你的 SSH 默认端口（22 或随机分配的端口）被封了**

搬瓦工购买后会自动分配一个随机的 SSH 端口，这个端口范围一般在 10000-65535 之间。这类端口被封，通常是因为你的服务在国内被"注意"到了——修改端口就能解决。

**第二种：特定服务端口（如 443、80）被封了**

这个稍微复杂一点。如果你的 SSH 端口本身通，但跑在某个端口上的服务不通（比如 443 端口死活不通），需要区分两种情况：

- 该端口没有任何服务在监听，显示关闭是**正常现象**
- 有服务在监听但仍不通，才是真的被封

还有一种常见误解：有人认为端口被封几次之后就会封 IP。这个逻辑**基本正确**——端口被反复盯上，IP 被封的概率确实会大幅提升。所以端口被封不要觉得无所谓，早发现早处理。

## 三、省钱修复方案：完全免费的端口修改教程

端口被封之后，你已经无法通过普通 SSH 登录——这是最头疼的地方。但搬瓦工提供了一个救命工具：**KiwiVM 控制面板里的在线 Shell（Root shell – interactive）**。

### Step 1：进入 KiwiVM 后台 VNC

1. 登录搬瓦工后台，进入 Services → My Services
2. 找到你的 VPS，点进去进入 KiwiVM Control Panel
3. 左侧菜单找到 **Root shell – interactive**，点 Launch
4. 输入 root 用户名和密码（密码输入时不会显示任何字符，属于正常现象）

### Step 2：修改 SSH 配置文件

登录后运行：

bash
vi /etc/ssh/sshd_config


找到 `Port` 那一行，按 `i` 进入编辑模式，把原来的端口号改成一个新的（建议选 10000-65535 之间没被用过的数字，比如 29725 或 38291）。

修改完成后，按 `Esc`，然后输入 `:wq` 保存退出。

### Step 3：重启 SSH 服务

bash
service sshd restart
# 或者
systemctl restart sshd


### Step 4：重新连接

用新端口重新 SSH 连接，一般立刻就能恢复。如果还是不行，可以在 KiwiVM 后台直接重启整台 VPS。

**整个过程零费用，5 分钟搞定。**
